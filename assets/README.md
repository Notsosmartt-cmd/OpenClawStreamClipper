# Asset libraries

Seed pack of CC0 / public-domain audio, video, and reaction-image assets used by
the AI editing-profile pipeline (see [[concepts/style-profiles]] in the wiki for
the design).

Binary files are NOT checked into git — they are regenerated by the fetch script.
Only this README, the per-folder `library.json` manifests, and the caption-style
templates are tracked.

## Layout

```
assets/
├── sfx/
│   ├── whoosh/        transitions, swipes (~0.3–1.0s)
│   ├── impact/        cuts, booms, hits (~0.5–1.5s)
│   ├── scratch/       record-scratches for comedy freeze-frames
│   ├── ding/          chimes, notifications (~0.3–0.8s)
│   └── riser/         build-ups (~1.5–4s)
├── music/
│   ├── hype/          high-energy beds (~140 BPM)
│   ├── funny/         playful (~110 BPM)
│   ├── emotional/     piano/strings (slow)
│   ├── storytime/     mid-tempo, lo-fi
│   └── tension/       news / hot-take beds
├── broll/
│   ├── travel/        flights, transit
│   └── general/       common-noun b-roll (extend via Pexels API)
├── memes/
│   ├── generic/       Twemoji reaction set (CC-BY 4.0)
│   ├── comedy/        comedy-context manifest (entries reference ../generic)
│   ├── hot_take/      same pattern
│   └── reactive/      same pattern
└── caption_styles/    ASS subtitle preset templates (Phase 3)
```

Each leaf folder has a `library.json`:

```json
{
  "version": 1,
  "entries": [
    {
      "file": "brwoosh1.mp3",
      "tags": ["whoosh", "transition", "swoosh"],
      "license": "CC0",
      "source": "https://archive.org/download/various-sound-effects/brwoosh1.mp3"
    }
  ]
}
```

## Seed it

```bash
# from repo root
python scripts/seed_libraries.py             # ~120 MB, 4–8 minutes
python scripts/seed_libraries.py --dry-run   # list without downloading
python scripts/seed_libraries.py --only sfx  # one category
python scripts/seed_libraries.py --only sfx,music
```

Re-runs are idempotent (existing files skipped, manifests merged).

## Add your own

Drop files anywhere under the matching category folder, then:

```bash
python scripts/seed_libraries.py --scan
```

That walks the tree, picks up new files, and rewrites every `library.json`.
Unknown files get a single best-guess tag derived from their filename. Edit
the manifest by hand to add richer tags — the LLM that picks cutaways reads
the `tags` list directly.

## Sources used

- **Internet Archive** — `various-sound-effects` (CC0), `allfreepdmusicbykuronekony4n` (FreePD/CC0), `PublicDomainCc0AirTravelStockVideoFootage` (CC0).
- **OpenGameArt.org** — CC0 single files + four 100-piece SFX packs by `rubberduck`.
- **Kenney.nl** — CC0 SFX packs (impact, interface, ui audio, sci-fi, digital).
- **Wikimedia Commons** — `Explosion-LS100155.ogg` (public domain).
- **Twemoji** — Twitter's emoji set (CC-BY 4.0). Attribution string is recorded in
  every manifest entry; no in-clip credit required by Twitter's CC-BY for
  derivative works under fair-use, but the manifest preserves it for paranoia.

## Sources NOT used (and why)

- **Pixabay / Pexels** — current TOS bars ML training and bulk extraction.
  Finished-clip output is fine but their license is not CC0. Use only via API
  with explicit per-clip retrieval if you set `PEXELS_API_KEY`.
- **Mixkit / Coverr** — proprietary licenses restrict redistribution.
- **BBC Sound Effects** — non-commercial only, unsafe for monetized clips.
- **Freesound.org** — even CC0 originals require OAuth login. Add
  `FREESOUND_API_KEY` and extend the script if you want them.
- **KnowYourMeme image hosts** — copyrighted reaction memes (Pepe, Spider-Man
  pointing, Hide-the-Pain Harold). User has accepted the IP risk: drop them
  into `assets/memes/<category>/` manually and run `--scan`. We don't hotlink
  KYM CDN URLs.

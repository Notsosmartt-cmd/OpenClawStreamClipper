# Session Handoff — 2026-07-09

One long session. Consolidated so a future agent doesn't reconstruct it from ~20 log entries.
Detail lives in the linked pages; this is the map + the load-bearing conclusions.

## 1. Speed campaign — CLOSED, with a complete measured ledger

The whole point: Stage 4 (moment detection, ~48 sequential LLM calls) is the run's dominant
cost. Every lever was measured; here's the final scorecard.

**Shipped DEFAULT-ON (production now):**
| Lever | Flag / mechanism | Win |
|---|---|---|
| Wave 1 #1/#2/#7 | audio-events cache, threaded scan, run-metrics | byte-identical (prior sessions) |
| **C3** reload hygiene | `stage2.py` skips the blanket model-unload on cached re-runs | no needless ~30-60 s reload |
| **C4** segment cache | `CLIP_SEGMENT_CACHE=1` (default on), kill `=0` | Stage 3 165 s → ~4 s on re-runs, byte-identical |
| **C1** cross-VOD prefetch | `CLIP_BATCH_PREFETCH=1` (default on), kill `=0` | overlaps next VOD's Stage 2 with render; contention +0.1% |
| **Moment-parallel** | `CLIP_PASSB_MOMENT_WORKERS=2` (default on), kill `=1` | **1.15× Stage 4** (~30 min/Lacy-class VOD) |

**Measured DEAD (tombstoned with numbers) — [[concepts/pipeline-speed-findings-2026-07]]:**
- **S1 speculative decoding = 8× SLOWER** (50→6 tok/s). The draft pays the same cross-vendor
  Vulkan per-pass coordination tax → pure overhead. GUI-only anyway on this runtime. §9/§9b.
- **C2b static-first prompt** — ~1 min/run, output-changing → DELETED (not shelved).
- **C6 grammar-constrained decoding** — 0 JSON-parse failures / 409 calls → CLOSED.
- **#3 vision-slot bench** noise; **#5 card-parallel** ~4%; **#6 vectorized scan** dominated by #2.

**THE load-bearing conclusion (findings §7 / §7b):** the Stage-4 "LLM floor" is now closed by
measurement. Parallelism 1.15×, spec-decode 0.12×, prefix-cache ~1 min, cards ~4% — every
software lever is measured and small, because on the cross-vendor Vulkan split (RTX 5060 Ti +
RX 6700 XT ≈ 28 GB pool) each forward pass is dominated by fixed NVIDIA↔AMD coordination that
batching/parallelism barely amortize (16% GPU util). **The only remaining Stage-4 speed levers
are hardware (a single ≥24 GB card that fits the model — also retro-unlocks spec-decode) or a
model-tier change (quality tradeoff).** Stop looking for config wins.

## 2. Thinking control — model-dependent; toggle + fail-fast guard shipped

- The pipeline's no-think stack (`/no_think` + `chat_template_kwargs.enable_thinking=False` +
  `reasoning_content` fallback + token budget) is CORRECT. Proven: **qwen3.6-35b-a3b = 0
  reasoning tokens** (control works).
- **gemma-4-26b-a4b-qat IGNORES the per-request kwarg entirely** (0 reasoning fresh-loaded,
  ~200 when its LM Studio TEMPLATE toggle is on). Since qwen honors the kwarg through the same
  LM Studio, it's gemma's chat template not wiring `enable_thinking`, not an LM Studio bug. So
  thinking control is **model-dependent**; the request kwarg reliably drives OFF on compliant
  models, and the model-level template is authoritative for gemma.
- **Shipped:** `scripts/lib/thinking.py` + `CLIP_ENABLE_THINKING` (default off, byte-identical)
  wired into all 6 LLM call sites + a dashboard checkbox; and `common.preflight_thinking()` —
  a Stage-4-start probe that ABORTS in ~1 s if the loaded model reasons on a trivial prompt
  (bypass `CLIP_ALLOW_THINKING_MODEL=1`). Turns a repeat of the BUG 67 wedge into a 1 s abort.
- Full detail + the correction arc: [[concepts/bugs-and-fixes#BUG 67]].

## 3. BUG 67 — the 8-hour wedge (two root causes, all fixed)

Owner's 3-VOD dashboard batch looked "stuck on Stage 4" for hours, then the PC crashed.
- **(A) gemma loaded with thinking ON** (owner's UI template toggle) → reasoned ~7997 tok per
  moment call → hit the 8000-tok cap → many empty answers, each chunk ~6 min. VOD 1 Lacy still
  finished (it's a **6.79 h VOD, ~61 Stage-4 chunks → ~5.9 h even mostly-healthy**, realtime
  0.82); VOD 2 Jynxzi was still grinding when the crash hit. Fix: reverted `models.json` (was
  an uncommitted gemma experiment) back to **qwen/qwen3.6-35b-a3b**.
- **(B) double-launch** — `start → stop → restart-dashboard-to-change-a-setting → play` ran two
  pipelines at once (the launch guard only checked in-memory state, blind across dashboard
  restarts/instances). Fix: `is_pipeline_running()` now checks the on-disk `pipeline.pid`
  marker (cross-process); a 2nd/restarted dashboard's play returns 409, and `api_stop` kills
  the marker pid even without an in-memory handle.

## 4. Moment-parallel — the gate/rubric case study

Arc worth remembering as a decision-making example: wired → A/B **1.15× / 4-10 overlap** →
REJECTED on a 1.4× bar → owner asked "why 1.4×?" (it was my judgment call) → owner chose a
different, legitimate philosophy ("take any speedup that doesn't hurt quality, esp. long VODs")
→ lowered bar to **1.1×**, made overlap **advisory** (a different-but-good clip draw is fine —
overlap ≠ quality) → owner **spot-checked** all 6 parallel-only clips (**4 good / 2 needs-work /
0 bad**) → PROMOTED default-on. 4 positive labels filed + frozen (owner pool now 20).

## 5. Companion punchline-only shorts — new feature (default off)

storytime/emotional (not rap/freestyle). [[concepts/clip-rendering]].

## 6. Doctrine crystallized this session (reuse these)

- **Gate by risk class, not one rule.** Byte-identity is the STRONGEST gate for cache/lifecycle/
  prefetch levers (a non-identical cache hit is a bug) and UNACHIEVABLE for LLM generation
  (batched FP reorder). For generation, use the **variance yardstick** (two serial temp-0.3
  re-runs of a VOD already share only 5/10 clips) + an owner spot-check. Overlap ≠ quality.
- **Owner rubric: default-off = RED.** A built-but-off lever is a zombie — promote it (with a
  gate) or delete it. Applied ruthlessly all session (C2b deleted, moment-parallel promoted).
- **Measure, don't theorize.** Every "obvious" win got a number: spec-decode's theoretical
  1.5-2.5× was 8× SLOWER; moment-parallel's textbook 2× was 1.15×. The rig's coordination floor
  beats intuition every time.
- **No zombie background tasks.** Every run bounded; kill cleanly; verify 0 stray procs.

## 7. Open items / where the real upside is now

- **Labels remain the #1 bottleneck** for the learning loop (ranker gates still honestly REJECT
  until miss-class labels accumulate). Owner pool = 20; near-miss review is the practical source.
- **Quality-lane themes from the 2026-07-09 clip review** (all orthogonal to speed):
  caption/title ↔ video mismatch (Stage 6 title-decoherence class); trim-vs-punchline precision;
  the owner repeatedly wants **SFX + visual effects at payoff beats** — the sfx-anchor lane is
  BUILT + validated but flag-gated (`CLIP_SFX`), a strong candidate for a real-post trial.
- Speed is now comfort, not capability — the channel-outcome levers are clip quality + labels.
